    Matriz Backward-From

    Objetivo

    Este documento visa ligar todos os requisitos elicitados durante a matéria de Requisito de Software aos seus respectivos métodos de elicitação e modelagem, utilizando o método de rastreabilidade Backward-From.


    Legenda

    Sigla Descricao
    QTN Questionário
    INT Instrospecção
    ST Storytelling
    RF Requisito Funcional
    RNF Requisito Não Funcional


    Categorias do Meta-Modelo

    Categoria Descrição
    Ambiental informações oriundas do contexto no qual a organização está inserida
    Desenvolvimento informações associadas aos diversos artefatos gerados ao longo do processo de desenvolvimento (artefatos de requisitos, diagramas, códigos, casos de teste e outros)
    Gerencial informações que auxiliam a gerência do projeto
    Organizacional informações pertencentes à organização (missão, objetivos e estratégias)

    ID Descricao Origem Elos
    03 Ter a opção de poder realizar seu login através de outras contas como facebook , google , etc, ou com seu email cadastrado. INT EF01
    04 Ter um campo para a senha cadastrada. INT EF02
    05 Mostrar um meio de recuperar a senha perdida. INT EF03
    06 Deixar claro para o usuário os Termos e Políticas do APP. INT EF04
    07 Perguntas específicas importantes para o app como: se ele é um profissional da saúde , se faz parte do grupo de risco e se é aluno de alguma instituição de ensino. INT EF05
    08 Opções de email,ou outras contas de uso do aluno, e senha INT EF06
    09 O sistema deverá perguntar ao usuário se ele está se sentindo bem o mal diariamente. INT EF07
    10 Na opção “Mal” deverá ser aberto um questionários abrangendo os sintomas sentidos e o período dos sintomas. INT EF08
    11 Deverá também conter questões importantes para o momento que atual de pandemia como: “saiu de casa nos últimos 14 dias?”,”teve contato com alguém?”. INT EF09
    12 O sistema deve garantir que o usuário responda apenas uma vez a pergunta. INT EF10
    13 O usuário poderá alternar sua resposta ao longo do dia. INT EF11

    16 O Mapa deverá informar de maneira clara as regiões e cidades contidas. INT EF14
    17 O sistema deverá nomear as ruas,vias e rodovias mostradas no mapa. INT EF15
    18 O sistema deverá demarcar porções no mapa indicando o grau de periculosidade delas através de cores. INT EF16
    19 Ao se clicar em uma dessas porções deverá ser informado ao usuário as quantidades de pessoas ,que responderam ao app, e a porcentagem de sintomáticos. INT EF17
    20 Deverá ser possível ao usuário navegar para regiões mais distantes de sua localização através do mapa exibido. INT EF18
    21 Deve ser possível alterar o nome de perfil. INT EF19
    22 Deve ser possível editar o nome que ficará disponível no perfil. INT EF20
    23 Deve ser possível mudar e-mail e senha referente a conta. INT EF21
    24 Deve existir um sistema de recompensa para outras universidades. QTN EF22
    25 Deve ser possível olhar os gráficos e dados estatísticos no próprio aplicativo . QTN EF23
    26 Deve ser corrigido os bugs dos registros diários de sáude. QTN EF24
    27 O usuário deve ser informado e atualizado constantemente sobre a situação da Covid-19 no país. QTN EF25
    28 O usuário deve ser notificado várias vezes ao dia, com mensagens diferentes, para reportar o seu status de saúde. QTN EF26
    29 Deve existir um mecanismo para verificar em quais regiões,principalmente hospitais, o usuário deva evitar caso tenha a necessidade de ir. QTN EF27
    30 Deve ser possível reportar bugs, e problemas técnicos pertinente ao sistema. QTN EF28
    31 O sistema tem que mostrar a quantidade de infectados em cada cidade. ST EF29
    32 O sistema tem que mostrar o total de infectados no DF. ST EF30
    33 O sistema tem que destacar as áreas mais afetadas. ST EF31
    34 O usuário tem que informar seu estado de saúde. ST EF32
    35 O usuário tem que informar sua cidade. ST EF33
    36 O sistema deve mostrar gráficos contendo o estado de saúde de cada usuário. ST EF34
    37 O sistema deve exibir um mapa mostrando as áreas mais afetadas. ST EF35
    38 O sistema poderia exibir a faixa etária dos infectados. ST EF36
